# words — тренажёр немецкой лексики

Персональный тренажёр слов: CSV-словари → адаптивные задания → прогресс в SQLite
с интервальным повторением (упрощённый SM-2).

Ядро — python-пакет `vocab` без внешних зависимостей (stdlib, Python 3.10+).
Поверх него работает `cli.py` с JSON-выводом — он же служит набором tools для
AI-агента в Telegram-боте (бот и агент подключаются отдельно).

## Быстрый старт

```bash
python3 cli.py sync        # загрузить слова из CSV в базу
python3 cli.py practice    # интерактивная тренировка в терминале
python3 cli.py stats       # посмотреть прогресс
```

## Структура проекта

```
data/                  словари (CSV) — источник правды о словах, подпапки разрешены
progress.sqlite3       прогресс обучения (создаётся автоматически)
cli.py                 CLI: JSON-команды + интерактивный режим
vocab/
  models.py            модели слов: Noun / Verb / VerbPrep
  storage.py           парсинг CSV, автоопределение типа строки
  db.py                SQLite: words, progress, tasks, reviews
  srs.py               SM-2: интервалы, ease, стадии 0–3
  scheduler.py         выбор слова и типа задания, проверка ответа
  exercises/
    base.py            нормализация и сверка ответов
    choice.py          множественный выбор (стадия 0)
    flashcard.py       перевод DE→RU (стадия 1) и RU→DE (стадия 2)
    cloze.py           пропуск в предложении (стадии 2–3)
    grammar.py         артикль/Plural, спряжение, предлог+падеж (стадия 3)
```

## Словари: формат CSV и добавление слов

Файлы — точка с запятой, без строки заголовка. Тип каждой строки определяется
автоматически по её содержимому, поэтому в одном файле можно смешивать типы:

| Тип | Распознаётся по | Колонки |
|---|---|---|
| Существительное | `der/die/das X (die Xe)` | слово; перевод; пример; произношение |
| Глагол | 22 колонки | те же 4 + 6×Präsens, 6×Perfekt, 6×Präteritum |
| Глагол с предлогом | `denken an + Akk` | слово; перевод; пример; произношение |
| Фраза (всё остальное) | — | слово; перевод; пример; произношение |

Для фраз («Fieber haben») работают карточки и выбор, но не cloze и грамматика —
планировщик сам подменяет тип.

**Добавить слово** — дописать строку в любой CSV в `data/` и выполнить `sync`.
**Добавить файл** — положить `.csv` в `data/` (можно в подпапку) и выполнить `sync`.

Директория словарей по умолчанию — `data/` рядом с `cli.py`; переопределяется
переменной окружения `WORDS_DATA`.

`sync` — это upsert по лемме, его можно запускать сколько угодно:

- новая лемма → слово добавляется и попадает в очередь новых;
- лемма есть, но поправили перевод/пример → запись обновляется, прогресс сохраняется;
- дубликат леммы в разных файлах → выигрывает первый файл по алфавиту;
- переименование самой леммы = удаление + новое слово (прогресс не переносится);
- удаление строки из CSV слово из базы **не** убирает — оно останется в ротации.

## Как устроено обучение

### Стадии и типы заданий

Стадия слова вычисляется из числа успешных повторов и текущего интервала
(`srs.stage`), и от неё зависит сложность задания:

| Стадия | Состояние | Типы заданий |
|---|---|---|
| 0 | новое / после ошибки | множественный выбор из 4 переводов |
| 1 | учится (1–2 повтора) | карточка DE→RU (узнавание) |
| 2 | закрепляется | карточка RU→DE (воспроизведение), пропуск в предложении |
| 3 | выучено | пропуск, грамматика: артикль/Plural, форма спряжения, предлог+падеж |

### Расписание повторений (упрощённый SM-2)

У каждого слова: `reps` (успешные повторы подряд), `ease` (лёгкость, старт 2.5),
`interval_days`, `due_at`. После ответа:

- **верно**: интервал 1 день → 3 дня → далее `интервал × ease`; ease подрастает
  или проседает в зависимости от качества ответа;
- **неверно**: `reps` сбрасывается в 0 (слово падает на стадию 0), ease −0.2,
  повтор через 10 минут.

Качество ответа (шкала SM-2): 5 — точный ответ в «продуктивном» задании
(RU→DE, cloze, спряжение), 4 — верный ответ, 3 — почти верно (опечатка,
пропущен артикль) — засчитывается, но интервал растёт медленнее, 1 — ошибка.

### Проверка ответов

`exercises/base.py` нормализует текст (регистр, пробелы, конечная пунктуация)
и допускает замену умляутов (`ae`/`oe`/`ue`/`ss`). Особые правила:

- **существительное без артикля** — засчитано с пометкой «нужен артикль»
  (качество 3); **неверный артикль** — ошибка;
- **глагол с предлогом без предлога/падежа** — засчитано с пометкой (качество 3);
- **опечатки** (similarity ≥ 0.85) прощаются в карточках и cloze, но **не в
  грамматике** — там «amüsierte» вместо «amüsiertest» это неверное окончание;
- в заданиях с вариантами можно отвечать номером варианта.

### Cloze: как находится слово в примере

Задание «пропуск» прячет изучаемое слово в примере из CSV, при этом слово в
предложении стоит в другой форме. Поиск учитывает: формы из таблицы спряжения,
единственное/множественное число, умляут (`halten`→`hält`), аблаут
(`bewerben`→`bewirbt`, `entscheiden`→`entschieden`), отделяемые приставки
(`vorbereiten`→`bereite … vor`) и причастия (`hören`→`gehört`).
Если форму найти не удалось, слово получает другой тип задания.

## CLI (JSON на stdout — это tools для агента)

```bash
python3 cli.py sync                              # импорт/обновление слов из CSV
python3 cli.py task                              # следующее задание (выбирает SRS)
python3 cli.py task --type cloze --word halten   # конкретное слово и/или тип
python3 cli.py answer <task_id> "ответ"          # проверить ответ, пересчитать расписание
python3 cli.py due                               # что пора повторить
python3 cli.py stats                             # стадии, точность за 7 дней, трудные слова
python3 cli.py word Salbe                        # полная карточка слова + прогресс
python3 cli.py practice                          # интерактивная тренировка
```

Типы заданий для `--type`: `choice`, `flashcard_de_ru`, `flashcard_ru_de`,
`cloze`, `grammar`.

База по умолчанию — `progress.sqlite3` рядом с `cli.py`; переопределяется
флагом `--db` или переменной окружения `WORDS_DB`.

### Жизненный цикл задания

1. `task` — планировщик берёт самое просроченное слово из очереди повторений
   (или новое, если повторять нечего), выбирает тип по стадии, генерирует
   задание и сохраняет его в таблицу `tasks`. Возвращает `task_id`, `prompt`,
   `options`/`hint`.
2. `answer <task_id> "..."` — ответ сверяется с сохранённым эталоном,
   обновляется расписание слова, пишется запись в журнал `reviews`.
   Возвращает `correct`, `expected`, `note`, `next_review_at`, новую стадию.
3. Повторный `answer` на тот же `task_id` отклоняется — задание одноразовое.

Состояние живёт в SQLite, поэтому между «выдать» и «проверить» процесс может
перезапускаться — важно для бота.

## Схема базы (`progress.sqlite3`)

| Таблица | Что хранит |
|---|---|
| `words` | реестр слов: лемма, тип, исходный файл, полные данные (JSON) |
| `progress` | по слову: reps, lapses, ease, interval_days, due_at |
| `tasks` | выданные задания: payload, эталон ответа, результат |
| `reviews` | журнал всех ответов: тип задания, верно/нет, качество, время |

CSV можно править как угодно — база подтянется через `sync`. Сбросить весь
прогресс = удалить `progress.sqlite3` и выполнить `sync` заново.

## Подключение AI-агента (tool calling)

Каждая CLI-команда — готовый tool. Минимальный набор:

| Tool | Команда | Зачем агенту |
|---|---|---|
| `get_next_task()` | `cli.py task` | выдать упражнение |
| `submit_answer(task_id, answer)` | `cli.py answer` | проверить и записать прогресс |
| `get_stats()` | `cli.py stats` | анализ прогресса, «что у меня западает» |
| `get_word_card(query)` | `cli.py word` | контекст для объяснений слова |
| `list_due()` | `cli.py due` | спланировать сессию |
| `sync_words()` | `cli.py sync` | подхватить новые CSV |

Проверка ответов детерминированная. Если агент считает ответ верным по смыслу,
а проверка его не засчитала — он видит `expected`/`note` и может выдать
повторное задание тем же словом (`task --word`), объяснив разницу.
